### **Slide 2: Objective**

### **Slide 3: Pipeline Overview**
- **Purpose**: Manage data flow and process recommendations.  
- **Pipelines**:  
  1. Store_Details  
  2. POS_Data  
  3. db_etl  
  4. Upload_to_Redis  
  5. Data Loading & Switching  

---

### **Slide 4: Store_Details Pipeline**
- **Purpose**: Fetch and update store-specific data.  
- **Process**:  
  - Extract details from EDAP table.  
  - Aggregate and save to Delta path.  
- **Key Notes**:  
  - Runs weekly (Monday) for 10 minutes.  
  - If it fails, previous week's data is used.  
- **Outcome**:  
  - Updated store details ensure accurate recommendations.

---

### **Slide 5: POS_Data Pipeline**
- **Purpose**: Create customer-product transactional data.  
- **Process**:  
  - Fetch product data via REST API.  
  - Combine with EDAP data to create transactional records.  
- **Key Notes**:  
  - Runs daily for 22 minutes.  
  - Inputs POS data into the db_etl pipeline.  
- **Outcome**:  
  - Generates accurate, real-time product data.

---

### **Slide 6: db_etl Pipeline**
- **Purpose**: Generate features for recommendations.  
- **Process**:  
  - Load POS_Data output into 9 notebooks.  
  - Create Delta path for processed features.  
- **Key Notes**:  
  - Runs weekly (Tuesday) for 3 hours.  
- **Notebooks**:  
  1. Product Pricing  
  2. Product Pairings (Compliment & Non-Compliment)  
  3. Context Similarity  
  4. Most Frequent SKU  
  5. Popular Products  
  6. User-Product Interaction  
  7. Validation  

---

### **Slide 7: Upload_to_Redis Pipeline**
- **Purpose**: Upload processed features to Redis cache.  
- **Process**:  
  - Excludes "User-Product Interaction" features.  
  - Transfers data from Delta path to Redis.  
- **Key Notes**:  
  - Runs weekly (Wednesday) for 15 minutes.  
- **Outcome**:  
  - Enables faster access to recommendation data.

---

### **Slide 8: Data Loading & Switching Pipeline**
- **Purpose**: Load "User-Product Interaction" data into SQL Server.  
- **Process**:  
  - Extract from Delta path and load into SQL.  
- **Key Notes**:  
  - Runs weekly (Wednesday) for 4 hours 50 minutes.  
- **Outcome**:  
  - Finalizes the recommendation data for GetRecs.

---
